# Donkeycar 시뮬레이터 기반 자율주행 데이터 관리 UI 구현

본 프로젝트는 Unity 기반 Donkeycar 시뮬레이터와 연동하여 자율주행 데이터를 효율적으로 확인, 정제하고 AI 모델 학습을 관리할 수 있는 Windows UI 어플리케이션 개발 계획을 담고 있습니다.

# 개발 계획
## 1. 프로젝트 개요

### Donkeycar 플랫폼 정의
* 소형 RC카 자율주행을 위한 Python 기반의 오픈소스 플랫폼입니다.
* 카메라 영상을 기반으로 주행 데이터를 학습하고 제어합니다.

### 우리 팀의 목표
* 시뮬레이터 주행 데이터 수집: Unity 기반 Donkeycar 시뮬레이터를 활용하여 실제 차량 없이도 카메라 영상과 조향값(Angle), 속도값(Throttle)이 결합된 주행 데이터를 안정적으로 수집합니다.
* 딥러닝 학습 파이프라인 수행: 수집된 catalog 데이터를 활용하여 Python 기반의 CNN 모델 학습 파이프라인을 실행하고 AI 모델을 구축합니다.
* C# WinForms UI 구현: 데이터셋의 확인, 불량 프레임 삭제, 학습 실행, 결과 시각화 등의 전 과정을 하나의 Windows 어플리케이션으로 통합하여 편리한 품질 관리 환경을 제공합니다.

---

## 2. 전체 동작 흐름

1. 시뮬레이터 주행
   * Unity Donkeycar 시뮬레이터 환경에서 사용자가 수동으로 RC카를 조종합니다.
2. 데이터 수집 및 자동 저장
   * 주행 중 카메라 이미지 파일과 각 프레임의 제어 레이블(angle / throttle)이 수집되어 catalog JSON 파일 형태로 자동 저장됩니다.
3. C# WinForms 데이터 관리 (품질 관리)
   * 저장된 데이터셋을 WinForms UI로 로드하여 불량 프레임 및 노이즈 데이터를 확인하고 즉각 삭제하여 데이터 품질을 높입니다.
4. AI 모델 학습 실행
   * UI에서 Train 버튼을 클릭하면 Python 학습 프로세스가 실행되며, 실시간 학습 로그와 성능 변화 추이를 Loss 그래프로 출력합니다.
5. 자율주행 실행 및 검증
   * 최종 학습 완료된 모델을 적용하여 시뮬레이터 또는 실제 차량에서 자율주행을 실행하고 성능을 검증합니다.

---

## 3. 시스템 구조

본 시스템은 사용자가 조작하는 C# WinForms UI 인터페이스와 백엔드의 Python 학습 엔진, 그리고 데이터가 생성되는 Unity 시뮬레이터가 유기적으로 연동되는 구조를 가집니다.

* UI에서 Python 실행 (C# to Python)
  * 사용자가 UI의 Train 버튼을 클릭하면, 내부적으로 'python manage.py train' 프로세스가 백그라운드에서 실행됩니다.
* Python에서 UI 로그 수집 (Python to C#)
  * Python 학습 엔진의 표준 출력(Standard Output)을 C# 프로그램이 실시간으로 수집하여 화면 내 ListBox 컨트롤에 로그를 출력합니다.
* 파일 기반 데이터 공유
  * 시뮬레이터가 생성한 catalog JSON 파일을 C# 어플리케이션이 직접 파싱하여 이미지 경로와 angle, throttle 레이블을 매칭하여 읽어옵니다.
* 시뮬레이터 연동
  * Unity Donkeycar Simulator가 자동으로 데이터를 저장하는 로컬 폴더 경로를 지정하여 데이터셋을 로드합니다.

## 4. UI 기능 및 컨트롤 구성 상세

어플리케이션은 크게 두 개의 탭(Data Manager, Training)으로 분리되어 직관적인 데이터 관리 환경을 제공합니다.

### 탭 1: Data Manager (데이터 확인 및 정제 화면)

* 폴더 열기 (Button 컨트롤)
  * 위치: Data Manager 탭 우측 상단
  * 동작: 클릭 시 FolderBrowserDialog(파일 탐색기)를 열어 데이터셋 폴더를 선택하고 로드합니다.
  * 결과: 폴더 내 이미지 파일 목록을 자동 수집하며, 슬라이더 최댓값과 프레임 카운터를 자동으로 초기화합니다. (지원 형식: JPG, PNG 등 일반 이미지 포맷)
* 이미지 출력 패널 (PictureBox 컨트롤)
  * 현재 선택된 프레임의 카메라 영상을 화면 중앙에 실시간으로 표시합니다. 이미지 파일이 없을 경우 빈 화면을 유지합니다.
* 프레임 번호 표시 (Label 컨트롤)
  * '프레임 표시' 레이블을 통해 현재 선택된 데이터의 인덱스를 실시간으로 출력합니다.
* 프레임 슬라이더 (TrackBar 컨트롤)
  * 최솟값 0부터 최댓값(전체 프레임 수) 범위 내에서 마우스 드래그를 통해 원하는 임의의 프레임 위치로 즉시 이동할 수 있습니다. 자동재생 시 화살표가 실시간 동기화되어 이동하며, 데이터가 삭제되거나 추가되면 최댓값이 재계산됩니다.
* 탐색 버튼 (Button 컨트롤)
  * 프레임 >> (다음 이동): 클릭 시 다음 프레임 데이터로 이동하며, 마지막 프레임에 도달하면 버튼이 비활성화됩니다.
  * 프레임 << (이전 이동): 클릭 시 이전 프레임 데이터로 이동하며, 첫 번째 프레임에서는 버튼이 비활성화됩니다.
* 프레임 삭제 (Button 컨트롤)
  * 현재 화면에 표시 중인 프레임을 데이터셋에서 즉시 제거합니다. 불량 이미지나 이상 조향값 등의 노이즈 데이터를 수동으로 정리하여 모델 학습 품질을 높이는 주된 목적을 가집니다. 삭제 후에는 자동으로 다음 프레임으로 인덱스가 이동하며 슬라이더 카운터가 재조정됩니다.
* 주행 레이블 시각화 (Button / Label 컨트롤)
  * 앵글 (Steering Angle): -1.0 (좌측 최대 조향)부터 +1.0 (우측 최대 조향) 범위를 가지며 중앙 0은 직진 상태를 의미합니다. 각 프레임의 조향 레이블을 시각화하여 표현하며, 모델이 예측해야 할 핵심 출력값입니다.
  * 속도 (Throttle): 0.0 (정지 상태)부터 +1.0 (최대 속도) 범위를 나타내며 음수 값은 후진 주행 데이터를 의미합니다.

### 탭 2: Training (AI 모델 학습 관리 화면)

* 모델 선택 (ComboBox 컨트롤)
  * CNN, LSTM 등 학습에 사용할 AI 모델 아키텍처를 드롭다운 메뉴로 선택합니다. 선택 즉시 해당 모델에 맞는 파라미터가 적용됩니다.
* Train 버튼 (Button 컨트롤)
  * 클릭 시 선택된 모델에 맞춰 데이터를 로드하고 전처리를 수행한 뒤, Python 기반의 학습 프로세스를 본격적으로 실행합니다. 학습이 완료되면 모델 파일이 지정된 경로에 자동 저장됩니다.
* 로그 출력 (ListBox 컨트롤)
  * 학습 프로세스로부터 들어오는 Epoch별 실시간 학습 진행 상황을 텍스트 형태로 화면에 출력하며, 프로세스 에러 발생 시 관련 오류 메시지를 상세히 표시합니다.
* Loss 그래프 (Chart 컨트롤)
  * 손실값(Loss)의 변화 추이를 바 차트 및 그래프 형태로 시각화하여 표현합니다. 이를 통해 사용자가 AI 모델의 학습 완성도 및 동향을 한눈에 파악할 수 있도록 돕습니다.

## 5. 구현 단계 로드맵

전체 개발 단계는 아래와 같이 총 8단계의 순차적 미션으로 진행됩니다.

1. catalog 파일 읽기: JSON 파일 구조를 분석하고 데이터를 파싱하는 기본 인프라 구축
2. 이미지 + 데이터 표시: UI 패널에 이미지 영상과 제어 수치를 동기화하여 매칭 출력
3. 프레임 슬라이더 구현: 전체 프레임을 자유롭게 탐색할 수 있는 트랙바 핸들러 연동
4. 그래프/자동재생 구현: 주행 데이터 수치의 그래프화 및 영상 형태의 타이머 재생 기능 개발
5. 데이터 필터링: 특정 조건이나 범위의 데이터를 선별할 수 있는 필터 조건부 처리
6. 데이터 삭제 기능: 불량 데이터를 물리적/논리적으로 제외하는 정제 기능 구현
7. 리스트 선택: 데이터셋 리스트 관리 및 개별 셋 선택 인터페이스 최적화
8. 학습 실행 및 연동: C# UI 환경에서 Python 백엔드 엔진을 호출하여 전체 학습 파이프라인 완성
